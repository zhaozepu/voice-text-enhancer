# 快速开始指南

## 🚀 启动程序

```bash
cd /Users/tal/Code/voice-text-enhancer
./run.sh
```

## ⌨️ 快捷键

**轻按右边 Option 键** 即可触发处理

- 单击即可，**无需长按**
- 长按（>0.5秒）不会触发，防止误操作

## 📝 使用步骤

1. **输入文本**
   - 使用豆包输入法语音转文字
   - 或直接输入/粘贴文本

2. **触发处理**
   - 光标停留在输入框内
   - 轻按一下**右边 Option 键**

3. **自动完成**
   - 工具自动全选 → 复制 → AI处理 → 替换
   - 显示动态进度："⠋ 正在处理文本，请稍候..."
   - 完成后显示："✓ 文本处理完成"

## ✨ 特点

- ✅ 单键触发，极简操作
- ✅ 自动全选，无需手动
- ✅ 不影响剪贴板（自动恢复）
- ✅ 实时进度提示
- ✅ 防误触设计

## 🔧 修改快捷键

编辑 `config.yaml`：

```yaml
hotkey:
  trigger: "right_option"  # 右 Option 键（默认）
  # trigger: "left_option"   # 左 Option 键
  # trigger: "right_cmd"     # 右 Command 键
  # trigger: "<cmd>+<alt>+space"  # 组合键
```

## 🎯 处理模式

当前默认：**expand**（扩展模式）
- 纠正语法错误
- 优化语句表达
- 适当添加背景信息

其他模式：
- **correct**：仅纠错
- **optimize**：纠错+优化
- **summarize**：总结精简

修改模式：编辑 `config.yaml` 中的 `active_prompt`

## 🐛 故障排除

### 快捷键无反应

1. 检查权限：
   - 系统设置 > 隐私与安全性 > **辅助功能** ✓
   - 系统设置 > 隐私与安全性 > **输入监控** ✓
   - 添加"终端"应用

2. 完全重启终端和程序

### 文本没有替换

1. 查看日志：`tail -f ~/.voice-text-enhancer/app.log`
2. 确认光标在输入框内
3. 检查是否有详细日志输出

### API 调用失败

1. 检查 `.env` 中的 `DEEPSEEK_API_KEY`
2. 检查网络连接
3. 查看日志中的错误信息

## 🔄 停止程序

按 `Ctrl+C` 退出

## 💡 最佳实践

1. **豆包输入法 + 右 Option** 完美组合
   - 按 Fn 开始语音
   - 松开 Fn 结束语音
   - 轻按右 Option 优化文本

2. **防止误触**
   - 只有快速单击才触发（<0.5秒）
   - 长按或误碰不会触发

3. **剪贴板安全**
   - 处理完成后自动恢复原剪贴板
   - 不影响你的正常复制粘贴
