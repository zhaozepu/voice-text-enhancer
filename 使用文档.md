## 💻 详细安装

### 系统要求

- **操作系统**: macOS 10.15 或更高版本
- **Python**: 3.8 或更高版本（通常 macOS 自带）
- **DeepSeek API Key**: 在 [platform.deepseek.com](https://platform.deepseek.com/api_keys) 免费注册获取

### 安装步骤

#### 方式一：自动安装（推荐）

```bash
cd /Users/tal/Code/voice-text-enhancer
bash scripts/install.sh
```

按照提示操作，脚本会自动完成：
- 创建虚拟环境
- 安装所有依赖
- 创建配置文件

#### 方式二：手动安装

```bash
# 1. 创建虚拟环境
python3 -m venv venv

# 2. 激活虚拟环境
source venv/bin/activate

# 3. 安装依赖
pip install -r requirements.txt

# 4. 配置文件
cp .env.example .env
cp config.example.yaml config.yaml
```

### 配置 API Key

编辑 `.env` 文件：

```bash
# 使用你喜欢的编辑器打开
nano .env
# 或
open -e .env
```

修改内容：
```bash
DEEPSEEK_API_KEY=sk-你的API密钥
```

**如何获取 API Key：**
1. 访问 https://platform.deepseek.com/
2. 注册/登录账号
3. 进入"API Keys"页面
4. 点击"Create API Key"
5. 复制密钥到 `.env` 文件

### 授予系统权限

⚠️ **非常重要** - 程序需要以下权限才能工作：

#### 1. 辅助功能权限 (Accessibility)

**步骤：**
1. 打开"**系统设置**"（System Settings）
2. 点击"**隐私与安全性**"（Privacy & Security）
3. 选择左侧的"**辅助功能**"（Accessibility）
4. 点击右下角的 **+** 号
5. 找到并添加"**终端**"（Terminal）应用
6. 确保终端旁边的开关是**打开**状态

#### 2. 输入监控权限 (Input Monitoring)

**步骤：**
1. 在"隐私与安全性"中
2. 选择左侧的"**输入监控**"（Input Monitoring）
3. 同样添加"**终端**"应用
4. 确保开关打开

#### 3. 权限验证

运行权限检查工具：
```bash
source venv/bin/activate
python3 check_permissions.py
```

如果看到 ✓ 表示权限正常。

**注意：** 授予权限后，必须**完全退出并重新打开终端**才能生效！

## 🎨 功能说明

### 核心功能

#### 1. 智能结构化 (Typeless 模式)

**自动识别文本类型并结构化：**

| 文本类型 | 输出格式 | 示例 |
|---------|---------|------|
| 多个要点 | 有序/无序列表 | `1. 第一点` `2. 第二点` |
| 主题内容 | 标题 + 列表 | `## 标题` `- 要点` |
| 单一段落 | 优化段落 | 流畅的句子 |
| 问答对话 | Q&A 格式 | `Q: 问题` `A: 答案` |
| 操作步骤 | 有序列表 | `1. 步骤一` `2. 步骤二` |
| 关键信息 | 粗体突出 | `**重要信息**` |

#### 2. 语法纠错

- ✅ 纠正错别字
- ✅ 修正标点符号
- ✅ 优化语句结构
- ✅ 统一书面表达

#### 3. 表达优化

- ✅ 口语化 → 书面语
- ✅ 冗长句子 → 简洁分点
- ✅ 重复内容 → 去重整合
- ✅ 模糊表达 → 清晰准确

#### 4. 剪贴板保护

- ✅ 自动保存原剪贴板
- ✅ 处理完成后自动恢复
- ✅ 不影响正常复制粘贴

#### 5. 实时进度提示

处理期间显示动态进度：
```
⠋ 正在处理文本，请稍候...
⠙ 正在处理文本，请稍候...
⠹ 正在处理文本，请稍候...
```

完成后显示：
```
✓ 文本处理完成
```

## ⚙️ 配置选项

### 配置文件位置

- **主配置**: `config.yaml`
- **环境变量**: `.env`

### 修改快捷键

编辑 `config.yaml`：

```yaml
hotkey:
  trigger: "right_option"  # 右 Option 键（默认）
```

**可选快捷键：**

| 配置值 | 说明 | 示例 |
|--------|------|------|
| `right_option` | 右边 Option 键 | ⌥ |
| `left_option` | 左边 Option 键 | ⌥ |
| `right_cmd` | 右边 Command 键 | ⌘ |
| `<cmd>+<alt>+space` | Cmd+Option+空格 | ⌘⌥␣ |
| `<cmd>+<shift>+<enter>` | Cmd+Shift+回车 | ⌘⇧↩ |

### 切换处理模式

编辑 `config.yaml`：

```yaml
active_prompt: "typeless"  # 当前模式
```

**可用模式：**

| 模式 | 说明 | 特点 |
|------|------|------|
| `typeless` | 智能结构化（推荐） | 自动识别并结构化，保持原意 |
| `correct` | 仅纠错 | 只修正错误，不改变表达 |
| `optimize` | 纠错+优化 | 优化表达，轻量结构化 |
| `expand` | 纠错+扩展 | 适当添加细节和背景 |
| `summarize` | 总结精简 | 提取要点，删除冗余 |

### 自定义 Prompt

在 `config.yaml` 的 `prompts` 部分添加：

```yaml
prompts:
  my_style: |
    你是一个文本处理助手。
    请按照以下规则处理：
    1. 纠正错误
    2. 优化表达
    3. 保持简洁
    直接返回处理后的文本。

active_prompt: "my_style"
```

### API 配置

编辑 `config.yaml`：

```yaml
api:
  base_url: "https://api.deepseek.com/v1"
  model: "deepseek-chat"
  timeout: 30  # 超时时间（秒）
```

### 日志配置

```yaml
logging:
  level: "INFO"  # DEBUG, INFO, WARNING, ERROR
  file: "~/.voice-text-enhancer/app.log"
```

### 通知配置

```yaml
notifications:
  enabled: true          # 是否启用通知
  show_processing: true  # 显示处理中通知
  show_completion: true  # 显示完成通知
  show_errors: true      # 显示错误通知
```

## 🔧 故障排除

### 问题 1: 快捷键无反应

**可能原因：**
- 权限未授予
- 程序未运行
- 快捷键冲突

**解决方法：**

1. **检查权限**：
   ```bash
   python3 check_permissions.py
   ```
   
2. **重新授予权限**：
   - 系统设置 > 隐私与安全性 > 辅助功能
   - 系统设置 > 隐私与安全性 > 输入监控
   - 添加"终端"应用

3. **完全重启**：
   - 退出程序（Ctrl+C）
   - 关闭终端窗口
   - 重新打开终端
   - 重新运行程序

4. **更换快捷键**：
   编辑 `config.yaml`，换一个不冲突的快捷键

### 问题 2: 文本没有被替换

**可能原因：**
- 光标不在输入框内
- 辅助功能权限未授予
- 延迟时间不够

**解决方法：**

1. **确认光标位置**：
   - 必须在输入框内有光标闪烁
   
2. **查看日志**：
   ```bash
   tail -f ~/.voice-text-enhancer/app.log
   ```
   
3. **增加延迟**：
   编辑 `src/core/text_processor.py`，将 `asyncio.sleep` 时间增加

### 问题 3: 提示"请先选中要处理的文本"

**可能原因：**
- 输入框为空
- 模拟 Cmd+C 失败

**解决方法：**

1. **确认有文本**：
   - 输入框中必须有内容
   
2. **手动测试**：
   - 在输入框中按 Cmd+A
   - 再按 Cmd+C
   - 看是否能复制

3. **查看详细日志**：
   ```bash
   tail -f ~/.voice-text-enhancer/app.log
   ```

### 问题 4: API 调用失败

**可能原因：**
- API Key 错误
- 网络问题
- API 额度用完

**解决方法：**

1. **检查 API Key**：
   ```bash
   cat .env | grep DEEPSEEK_API_KEY
   ```
   
2. **测试 API**：
   ```bash
   curl https://api.deepseek.com/v1/models \
     -H "Authorization: Bearer sk-你的API密钥"
   ```
   
3. **查看错误信息**：
   ```bash
   tail -n 50 ~/.voice-text-enhancer/app.log
   ```

### 问题 5: 粘贴的是旧内容

**可能原因：**
- 剪贴板时序问题
- 延迟不够

**解决方法：**

1. **查看日志验证**：
   - 日志中会显示"验证剪贴板，匹配: True/False"
   
2. **增加延迟**：
   编辑 `src/core/text_processor.py`：
   ```python
   await asyncio.sleep(0.8)  # 从 0.5 增加到 0.8
   ```

## ❓ 常见问题

### Q: 需要联网吗？
**A:** 是的，需要联网调用 DeepSeek API 进行文本处理。

### Q: 处理一次要多久？
**A:** 通常 2-5 秒，取决于：
- 文本长度
- 网络速度
- API 响应时间

### Q: 会保存我的文本吗？
**A:** 不会。所有处理都是实时的，不保存任何文本内容。

### Q: 会消耗多少 API 额度？
**A:** 每次处理约 100-500 tokens，具体取决于文本长度。
DeepSeek 有免费额度，日常使用足够。

### Q: 支持哪些应用？
**A:** 理论上支持所有文本输入框，包括：
- 微信、钉钉、Slack
- 备忘录、Word、Pages
- 浏览器输入框
- VS Code、其他编辑器

### Q: 支持多语言吗？
**A:** 支持。DeepSeek 可以处理中文、英文等多种语言。

### Q: 可以处理多长的文本？
**A:** 建议单次处理不超过 2000 字，过长的文本可能：
- 处理时间变长
- 消耗更多 API 额度

### Q: 如何更新程序？
**A:** 
```bash
cd /Users/tal/Code/voice-text-enhancer
git pull  # 如果使用 git
# 或重新下载最新版本
```

### Q: 可以在多台电脑使用吗？
**A:** 可以，只需：
1. 复制整个项目文件夹
2. 在新电脑上重新配置 API Key
3. 授予系统权限

### Q: 快捷键为什么选右 Option？
**A:** 
- 单键操作最简单
- 右 Option 不常用，不易冲突
- 位置方便，不影响输入

可以改成其他键，编辑 `config.yaml` 即可。

### Q: 如何查看处理历史？
**A:** 程序不保存历史。如需查看操作日志：
```bash
cat ~/.voice-text-enhancer/app.log
```
