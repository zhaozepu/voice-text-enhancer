# Typeless 模式示例

新的 `typeless` 模式模仿 Typeless 的智能处理逻辑：
- **智能识别文本类型**
- **自动结构化输出**
- **保持原意，优化表达**

## 📋 示例对比

### 示例 1: 多个要点 → 自动列表化

**输入**（语音转文字）：
```
今天要做三件事首先要去超市买菜然后要给客户打电话确认订单最后要写一份项目报告
```

**输出**（自动结构化）：
```
今天要做三件事：

1. 去超市买菜
2. 给客户打电话确认订单
3. 写一份项目报告
```

---

### 示例 2: 会议记录 → 标题+列表

**输入**：
```
今天开会讨论了产品规划主要有几个重点第一个是用户体验优化要简化注册流程第二个是性能提升要减少加载时间第三个是增加新功能包括消息推送和数据导出
```

**输出**（智能结构化）：
```
## 产品规划会议要点

### 用户体验优化
- 简化注册流程

### 性能提升
- 减少页面加载时间

### 新功能开发
- 消息推送
- 数据导出
```

---

### 示例 3: 单一段落 → 优化表达

**输入**：
```
这个方案我觉得还行但是有些地方需要改一下比如说时间安排太紧了还有预算也不太够
```

**输出**（段落优化）：
```
这个方案整体可行，但有两个方面需要调整：**时间安排过于紧凑**，**预算不足**。建议重新评估资源配置。
```

---

### 示例 4: 问答对话 → Q&A 格式

**输入**：
```
用户问我们的产品支持哪些平台我回答说目前支持iOS安卓和网页版后续会推出桌面应用
```

**输出**（Q&A 结构）：
```
Q: 产品支持哪些平台？

A: 目前支持以下平台：
- iOS
- Android
- 网页版

计划推出桌面应用。
```

### 示例 5: 操作步骤 → 有序列表

## 🎯 Typeless 模式的特点

### ✅ 会做的事情
1. **智能识别文本类型**
   - 列表内容 → 使用 `-` 或 `1.`
   - 有主题的内容 → 使用 `##` 标题
   - 关键信息 → 使用 `**粗体**`
   - 问答 → 使用 `Q:` `A:` 格式

2. **优化表达**
   - 纠正语法错误和错别字
   - 口语化 → 书面语
   - 冗长句子 → 简洁分点
   - 重复内容 → 去重整合

3. **保持原意**
   - 不添加原文没有的内容
   - 不过度扩展
   - 保留核心信息

### ❌ 不会做的事情
1. 不添加 "以下是..." 等前缀
2. 不过度扩展内容
3. 不改变核心意思
4. 不对单句进行过度结构化

## 🔧 使用建议

### 适合场景
- ✅ 语音转文字后的整理
- ✅ 会议记录整理
- ✅ 待办事项清单
- ✅ 操作步骤说明
- ✅ 问答对话记录
- ✅ 笔记整理

### 切换模式
如果觉得结构化过度，可以切换到其他模式：

编辑 `config.yaml`：
```yaml
active_prompt: "correct"   # 仅纠错，不结构化
# active_prompt: "optimize"  # 纠错+优化，轻量结构化
# active_prompt: "typeless"  # 智能结构化（默认）
```

## 💡 最佳实践

### 豆包输入法 + Typeless 完美组合

1. **语音输入** 
   - 按住 Fn 说话
   - 松开 Fn 结束

2. **自动优化**
   - 轻按右 Option 键
   - 等待 2-5 秒

3. **智能输出**
   - 自动纠错
   - 自动结构化
   - 专业表达

### 示例工作流

```
语音："今天要完成三件事第一写文档第二开会第三写代码"
  ↓
豆包转文字："今天要完成三件事第一写文档第二开会第三写代码"
  ↓
轻按右 Option
  ↓
Typeless 处理：
"今天要完成三件事：

1. 写文档
2. 开会
3. 写代码"
```

完美！清晰！专业！
